## Consolidation notes

- Catalogue grid/list/filter/loading/no-results are one route model with URL search state, not separate data models.
- Processing active/completed/failed/retrying are states of one job-detail contract, not separate domain models.
- Upload default/in-progress/validation-error share one upload workflow model.
- Approval/correction modal screens are overlays under approval or document routes.
- Reader and admin compact shell screens are responsive variants of layouts.
- Reference folders feed the design-system and UI decision docs rather than product routes.
